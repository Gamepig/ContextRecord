# 產品背景 (Product Context)

## 專案存在的目的
本專案旨在為 AI 應用程式提供一個可靠的後端服務，專門用於紀錄和管理使用者與 AI 之間的對話內容。這是一個基礎且關鍵的組成部分，支援 AI 應用程式的狀態管理、歷史查詢和未來的分析需求。

## 解決的問題
- **對話歷史的持久化：** 許多 AI 互動是無狀態的，難以追蹤完整的對話上下文。本專案解決了如何持久化儲存對話歷史的問題。
- **對話內容的搜尋與檢索：** 在需要回溯或分析歷史對話時，能夠高效地搜尋和檢索相關內容至關重要。本專案提供了實現此功能的基礎架構和 API。
- **支援未來的 AI 增強功能：** 透過使用 PostgreSQL 和考慮整合 pgvector，本專案為未來實現更進階的 AI 搜尋功能 (如語義搜尋) 奠定了基礎。
- **開發與部署的一致性：** 透過 Docker 化，解決了在不同開發、測試和生產環境中部署應用程式時可能遇到的環境不一致問題。
- **即時互動的需求：** 使用 SSE 解決了需要伺服器主動向客戶端推送即時對話更新或其他事件的需求。

## 目標使用者
- 建立需要對話歷史紀錄功能的 AI 應用程式開發者。
- 需要分析或利用 AI 互動數據的數據科學家或分析師 (未來功能)。

## 成功標準
- 核心 API (紀錄與基本搜尋) 功能正常且穩定。
- SSE 連接能夠穩定建立並推送訊息。
- 應用程式能夠成功建置為 Docker 映像檔並運行。
- CI/CD 工作流程能夠自動執行程式碼檢查、測試和部署。
- 專案文件清晰且易於理解。
